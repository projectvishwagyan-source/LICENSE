## ğŸ“œ Contribution Rules

### 1. Code Standards
- We follow **PEP 8** for Python code.
- Ensure your code is well-commented, especially logic related to RAG and Embeddings.
- Use meaningful commit messages (e.g., `feat: add Marathi tokenizer` or `fix: resolve Pinecone connection timeout`).

### 2. Data Integrity (Crucial!)
Since Vishwagyan is used for UPSC, **accuracy is everything**.
- If you are contributing data, it must be from an **official government source** (NCERT, PIB, etc.).
- No unverified blog posts or biased articles allowed.

### 3. Pull Request (PR) Process
- Ensure your branch is up-to-date with the `main` branch.
- Submit a PR with a clear description of what you changed and *why*.
- Wait for the maintainers to review your code. We might suggest some changes before merging.
